07 · Responsive

Comportamiento responsivo

La sidebar tiene dos comportamientos: sticky en desktop y drawer off-canvas en mobile/tablet.

Desktop ≥ 961px
  • Ancho fijo: 264px (--sidebar-w)
  • position: sticky · top: 56px
  • Altura: calc(100dvh - 56px)
  • Siempre visible, no hay trigger
  • overflow-y: auto con scrollbar custom
  • Border right: 1px solid var(--line)
Mobile / Tablet ≤ 960px
  • position: fixed · full height
  • Oculta por defecto: translateX(-100%)
  • Se activa con .open → translateX(0)
  • Triggered por .hamburger-btn en el topbar
  • Overlay backdrop blur al abrirse
  • Se cierra al hacer clic en el overlay o en un ítem
  • Tecla Escape cierra el drawer

09 · Guías

Uso correcto

Lineamientos de diseño para mantener la coherencia del componente.

Hacé

Usá nav-group para módulos con varios sub-artículos (3 o más). Los grupos dan estructura y evitan la saturación visual.

No hagas

No pongas más de 8 ítems hijos en un mismo grupo. Si un módulo tiene muchos sub-temas, dividilo en secciones o usá un segundo nivel de agrupación.

Hacé

Usá nav-link (sin hijos) para páginas independientes y de alto nivel como "Inicio" o "FAQ" que no pertenecen a ningún módulo.

No hagas

No omitas los iconos en los nav-group-header. Los íconos son clave para la identificación rápida del módulo, especialmente cuando la sidebar está a punto de cerrarse.

Hacé

Usá section labels para separar grandes bloques de navegación (ej: "Módulos" vs "Soporte"). No más de 4 secciones en una misma sidebar.

No hagas

No hardcodees colores o tamaños fuera de los tokens. Todo cambio de tema (dark/light) depende de que los valores estén en variables CSS. Un valor fijo rompe el theming.

Scroll de la sidebar La sidebar tiene overflow-y: auto. Si el contenido excede la altura visible, aparece un scrollbar custom de 5px. En mobile, el scrollbar también aplica dentro del drawer.

11 · Figma

Implementación en Figma

Guía paso a paso para construir el componente Sidebar en Figma siguiendo los tokens del sistema.

1
Variables de color
En tu Variable Collection (grupos Dark/Light), mapeá todos los tokens de la tabla de colores de este documento. Asegurate de que bg-elev, bg-soft, cyan y cyan-soft estén disponibles.
2
Nav Icon (átomo base)
Creá un componente Nav/Icon: frame 28×28px, corner radius 6px, background bg-soft. Agregá un slot de icono SVG 13×13px centrado. Variantes: default y active (background cyan-soft-2, color cyan).
3
Nav Child (átomo)
Frame de ancho fill, auto height. Padding 7px 10px 7px 46px. Text style: Body/12-Regular. Variantes: Default, Hover, Active. En Active: color cyan, fill cyan-soft, font-weight 500.
4
Nav Group Header (molécula)
Auto layout horizontal, gap 8px, padding 8px 10px, corner 10px. Instancia de Nav/Icon + texto (fill) + chevron 14×14. Variantes: Closed, Hover, Open. En Open: rotar chevron 90° y cambiar Nav/Icon a variante active.
5
Nav Group (organismo)
Auto layout vertical. Instancia de Nav Group Header + frame hijos (Nav Child repetidos). Añadí propiedad booleana Expanded: true/false para mostrar/ocultar el frame de hijos. Usá Auto layout con clip content.
6
Sidebar (template)
Frame 264×altura-variable. Auto layout vertical, padding 16px 10px. Background bg-elev, border right 1px line. Apilá: Section Label + Nav Groups + Dividers + Nav Links según el contenido.
Prototipo interactivo Para prototipar el collapse en Figma, usá Smart Animate entre variantes del Nav Group con duración 250ms y easing Custom (0.4, 0, 0.2, 1).
